# Grid Coffee 点单小程序还原版

这是基于线上生产环境可见页面、手机镜像和截图重建的前端工程。

线上默认打开点单小程序预览；后台编辑界面通过 `?admin=1` 进入，后台中的内容会驱动小程序预览。

## 运行

```bash
pnpm install
pnpm dev
```

本地地址：

```text
http://127.0.0.1:5173/
```

后台编辑入口：

```text
http://127.0.0.1:5173/?admin=1
```

## GitHub Pages 部署

项目已包含 GitHub Actions 部署配置：

```text
.github/workflows/deploy.yml
```

推送到 `main` 后会自动构建 `dist` 并发布到 GitHub Pages。Vite 已使用相对资源路径，适配 GitHub Pages 的仓库子路径部署。

## 已实现

- 后台：OGS 风格商户管理界面，支持编辑首页、菜品、套餐、商城、订单样例、会员、JSON 备份。
- 首页：活动 Hero、会员进度、到店自取/咖啡外送、快捷入口、底部导航。
- 点单页：自取/外卖切换、门店信息、搜索入口、分类栏、商品列表、打烊状态。
- 商品交互：普通单品规格弹窗、依云冷萃双杯组合套餐页、早餐任意搭组合页。
- 商城页：商城 Hero、分类 Tab、商品网格、售罄状态、购物袋计数。
- 订单页：订单中心、历史订单卡、订单详情。
- 会员页：会员卡、权益入口、设置入口。

## 后台数据

- 默认内容来自 `src/data.ts`。
- 运行后编辑内容会保存在浏览器 `localStorage` 中，不会写入线上 OGS 后台。
- 后台的“数据备份”页可以导出/导入 JSON，方便后续迁移到真实接口或数据库。

## 当前边界

- 当前版本是前端还原和本地模拟，没有接真实微信登录、支付、订单或门店接口。
- 当前生产环境截图显示门店已打烊，所以点单加入购物车/提交按钮按原页面保持灰置。
- 商品图片来自本次采集截图裁切，后续如果能拿到原始素材，可替换 `public/assets`。
